# Troubleshooting

## 1. STM32 JTAG 핀 충돌 문제

### 문제
STM32F103RB 프로젝트에서 Keypad 입력 일부가 정상적으로 동작하지 않는 문제가 발생했습니다.

### 원인
PB3, PB4 핀이 기본적으로 JTAG 기능에 할당되어 있어 GPIO 입력으로 사용할 수 없는 상태였습니다.

### 해결
AFIO 설정을 통해 JTAG 기능을 비활성화하고 해당 핀을 GPIO로 사용할 수 있도록 수정했습니다.

### 배운 점
임베디드 시스템에서는 코드 로직뿐 아니라 MCU 핀의 기본 기능, 리맵 설정, 주변장치 클럭 설정까지 함께 확인해야 합니다.

## 2. Dot Matrix 핀 결함 대응

### 문제
Dot Matrix 일부 핀이 정상 동작하지 않아 기존 8x16 출력 구조를 그대로 사용할 수 없는 문제가 발생했습니다.

### 해결
출력 구조를 8x15 형태로 우회하여 핵심 표시 기능을 유지했습니다.

### 배운 점
하드웨어 결함이나 제약이 발생했을 때, 전체 기능을 포기하기보다 요구 기능을 만족할 수 있는 대체 구조를 설계하는 것이 중요합니다.

---

## 3. 점자 검증 기준 조정

### 문제
점자 표현 범위의 한계 때문에 동일한 점형에서 다양한 한글 해석군이 발생할 수 있었습니다.

### 해결
문자 해석 자체보다 제작된 점형이 목표 패턴과 일치하는지를 판단하는 방향으로 검증 기준을 조정했습니다.

### 배운 점
검증 시스템에서는 무리한 해석보다 판정 가능한 단위를 명확히 정의하는 것이 중요합니다.

---

## 4. Isaac Sim QR 및 박스 방향 문제

### 문제
QR 코드가 카메라에 정상적으로 보이지 않거나, 박스 방향에 따라 인식이 불안정해지는 문제가 발생했습니다.

### 해결
로봇 내장 카메라 방식 대신 월드 고정 카메라 방식으로 전환하고, QR 코드가 보이는 면을 기준으로 박스 방향을 고정했습니다.

### 배운 점
시뮬레이션 환경에서도 센서 위치, 객체 방향, 인식 조건이 전체 시스템 안정성에 큰 영향을 미친다는 점을 경험했습니다.

---

## 5. 단계별 통합 관리

### 문제
QR 인식, 박스 생성, 컨베이어 흐름, 로봇 분류 동작이 한꺼번에 결합되면서 오류 원인을 찾기 어려운 상황이 발생했습니다.

### 해결
단일 로봇 동작 검증, QR 인식 검증, 박스 방향 검증, 2대 로봇 통합 순서로 단계별 테스트를 진행했습니다.

### 배운 점
복잡한 시스템은 기능을 작게 나누어 검증하고, 이후 통합 범위를 넓히는 방식이 안정적입니다.

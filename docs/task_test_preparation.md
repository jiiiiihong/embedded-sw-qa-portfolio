# Task Test Preparation

## 예상 과제 방향

Embedded SW QA Engineer 직무의 과제테스트는 단순 알고리즘 코딩테스트보다 직무 상황 기반 분석 과제일 가능성이 높습니다.

예상 유형은 다음과 같습니다.

## 1. 품질 데이터 분석 과제

### 예시
결함 데이터 CSV가 주어지고, 모듈별 결함 수, 심각도별 분포, 재발률, 주요 원인을 분석한 뒤 개선 방안을 제시하는 과제입니다.

### 준비 항목
- Python
- pandas
- matplotlib
- 결함 유형 분류
- 품질 지표 정리
- 분석 결과 보고서 작성

## 2. SW 품질 프로세스 점검 과제

### 예시
요구사항, 설계서, 테스트케이스 일부가 주어지고, A-SPICE 관점에서 추적성 누락 또는 검증 부족 항목을 찾는 과제입니다.

### 준비 항목
- 요구사항 → 설계 → 구현 → 테스트 흐름 이해
- 추적성 개념
- 테스트케이스 누락 판단
- 변경 이력 및 결함 조치 흐름 이해

## 3. 임베디드 SW 코드 리뷰 과제

### 예시
C 코드 일부 또는 테스트 로그가 주어지고, 예외 처리 부족, 경계값 테스트 누락, 상태 전이 오류 가능성을 찾는 과제입니다.

### 준비 항목
- C 기본 문법
- 인터럽트 처리
- 상태 머신
- 경계값 테스트
- 테스트 로그 분석

## 4. 필드 이슈 원인 분석 과제

### 예시
특정 차량 SW 기능에서 반복 이슈가 발생했고, 원인 분석 및 재발 방지 방안을 작성하는 과제입니다.

### 준비 항목
- 5 Why
- Fishbone Diagram
- 재현 조건 정리
- 단기 조치와 장기 대책 구분
- 재발 방지 체크리스트 작성

## 기본 개념 정리

### Functional Safety
차량 SW가 고장 나더라도 사람이 위험해지지 않도록 위험을 분석하고 안전 요구사항을 정의하는 개념입니다. 자동차 분야에서는 ISO 26262와 연결됩니다.

### A-SPICE
자동차 SW 개발 프로세스가 요구사항, 설계, 구현, 테스트, 검증 흐름을 갖추고 있는지 평가하는 기준입니다.

### AUTOSAR
자동차 ECU 소프트웨어를 표준화된 구조로 개발하기 위한 플랫폼입니다. Application Layer, RTE, Basic Software, MCAL 등의 계층으로 구성됩니다.
